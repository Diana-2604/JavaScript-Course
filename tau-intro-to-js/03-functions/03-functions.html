Functions in JavaScript

Functions have a few goals:

  • Help you organize your code for readability
  • Allow you to take blocks of code and make them easily repeatable, thus reducing errors

We can package our reusable code into a function, and have that function be the only place we have to make the update if changes are needed.

Create a new JavaScript file

In the same file folder, create a second file with the extension .js. This will be our external JavaScript file.
When add src="./04-functions.js" to the script tag in yout html to link these two files together.

Write a function

In the .js file, we have the function keyword, we have the name of the function, we have the inputs or parameters inside the ()
Then inside the {}, the logic, and then whatever the output is.

When we're saying return, this means give the output back to wherever it's being called.

function sum(a, b) {
return a+b;
}

Call the function

You have to use the parentheses () when you want the function to do its job.

console.log(sum(10, 10));
The function output can also be stored into a variable.

const result = sum(5, 25);

Anonymous functions

These are functions that have the same rules - they need an input, output and logic, but we don't give these functions a name, hence anonymous.

You'll see this most commonly when you need to pass a function as a callback. In JavaScript, callbacks are functions that can be used as inputs to another function.

A great example of this is the setTimeout function. setTimeout() takes two inputs, a function, and a time in milliseconds.

The function is the behavior that should happen, and the time in milliseconds is how long we should wait for that behavior to happen.

setTimeout(function () {
console.log("Hey! You rock!!!");
}, 1000);
The setTimeout will call our function that we passed in our anonymous function after one second.

Arrow functions

Arrow functions are just like standard functions, except for there's a bit of syntactic difference.

The first thing that you'll see with an arrow function is that - well - you don't need the function keyword.

These parentheses still represent our input list, but between the parentheses () and the curly braces {}, you have to put an arrow.

Now to make an arrow, you do the equal sign and then a greater than sign - =>.

If you have one line of code only, you can drop the curly braces {}.

This is how the final arrow function looks:

setTimeout(() => console.log("Hey! You rock!!!"), 1000);
In arrow functions, if you have one line of code, the arrow function assumes that that is your return value.

With arrow functions, if you want to use more than one line of code, you have to bring the curly braces {} back and then you will have to use the return keyword as well.
